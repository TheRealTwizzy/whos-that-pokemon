# Project Policies

## Mandatory Skill Invocation
- **UI/UX & Game Loop**: MUST invoke `$game-studio`.
- **File/Stack Operations**: MUST invoke `$gstack`.
- **Complex Logic & Refactoring**: MUST invoke `$superpowers`.

## Workflow Enforcement
- **Multi-Step Tasks**: MUST use **Hierarchical Orchestrator** pattern (Spawn -> Wait -> Merge).
- **Critical Changes**: MUST implement **Debate/Critique Loop** before committing.
- **Data Processing**: MUST use **Fan-In/Fan-Out** for parallel execution.

## Definition of Done
- No PR is complete without a `$superpowers` review pass.
- All UI changes must be validated by `$game-studio`.
